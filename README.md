# 🃏 マッチング管理

カードゲーム交流会・対戦会のためのブラウザ完結型マッチング管理アプリです。  
サーバー不要・インストール不要。ブラウザだけで動作します。

## 🌐 デモ

**[→ アプリを開く](https://soichirow.github.io/matching-kanri/)**

---

## 主な機能

| 機能 | 説明 |
|------|------|
| 👤 プレイヤー管理 | 名前・グループの登録、ステータス管理（待機／対戦中／除外） |
| 🃏 テーブル管理 | テーブルの追加・配置、キャンバス上でドラッグ移動 |
| 🎲 マッチング生成 | 同グループ・対戦済みを避けるスコアベース自動マッチング |
| 🔮 仮マッチング | 確定前にテーブルへプレビュー表示、個別確定/取消 |
| 📋 対戦履歴 | プレイヤーごとの対戦履歴ダイアログ |
| 📊 レイアウト管理 | 部屋オブジェクト（モニター・入口など）の配置 |
| 📥 Excelから一括入力 | TSV貼り付けによるプレイヤー・テーブルの一括登録 |
| ↩️ Ctrl+Z で元に戻す | 最大30件のアンドゥスタック |
| 💾 自動保存 | localStorage に自動保存（ページを閉じてもデータが残ります） |

## 使い方

### 基本フロー

```
1. プレイヤーを追加（＋ プレイヤー / Excelから一括入力）
2. テーブルを追加（＋ テーブル）
3. 🎲 マッチング生成 → テーブルに仮配置される
4. 確定（各テーブル）またはすべて確定
5. 対戦終了 / 解散 で次のラウンドへ
```

### ショートカット

| キー | 動作 |
|------|------|
| `Ctrl+Z` | 元に戻す |
| ダブルクリック | プレイヤー名・グループを編集 |
| `Shift+クリック` | プレイヤーを範囲選択 |

### マッチング対象の絞り込み

- **プレイヤーにチェック** → チェックしたプレイヤーだけを対象にマッチング
- **テーブルを選択** → 選択したテーブルだけに割り当て
- 両方指定すると「そのプレイヤーをそのテーブルへ」となる

## ファイル構成

```

src/
  store.js          # localStorage CRUD
  matching.js       # マッチングアルゴリズム（スコアベース）
  layout.js         # ドラッグ&ドロップ
tests/
  store.test.js
  matching.test.js
  layout.test.js
```

## 開発

```bash
# テスト実行
npm test

# テスト（1回のみ）
npm run test:run

# Lint
npx eslint src/store.js src/matching.js src/layout.js
```

### マッチングアルゴリズム

`src/matching.js` の `generateMatching()` が中核です。

- **ペナルティスコア**で最適な組み合わせを探索
  - 同グループ対戦：+5点
  - 対戦済み1回ごと：+10点
- 20人以下のときは全組み合わせ探索、それ以上はランダムサンプリング

## データについて

- データはブラウザの **localStorage** に保存されます
- ファイルを閉じてもデータは残ります
- 設定 → データクリア で削除できます
- **プライベートブラウジングではデータが消えます**
